Philosophy

Why Chi exists.

Unix systems are powerful, stable, and free. They should also be beautiful, personal, and alive.

The care gap.

We've used Unix-based systems for years. We love what they stand for — openness, freedom, ownership. But somewhere along the way, the visual layer got left behind.

The people building interfaces and the people designing experiences became two different worlds. And users — real, everyday people — got caught in the middle.

Not because the technology wasn't there. But because design and engineering never shared a room.

What we believe.

At Smart, we believe the future of Unix-based systems should not feel outdated, overly technical, or difficult for new users to approach.

We believe these systems can be powerful and beautiful, flexible and welcoming.

We believe open-source software can feel warm, polished, and emotionally resonant.

Chi exists to help move that future forward.

Not a criticism. A contribution.

We don't build Chi to replace what exists. The foundations of open-source software are extraordinary — decades of brilliant engineering that changed the world.

We build Chi because we believe those foundations deserve a visual layer that matches their quality. Because the way software looks and moves is how it communicates with the person using it.

Beauty isn't superficial. It's how software says "I was made with you in mind."

Design

Principles that breathe.

Every decision in Chi traces back to five core beliefs about how software should work.

01 Beautiful by default

If the developer writes zero styling code, the app should still look intentional. Chi components ship with considered spacing, typography, colour, and motion. Beauty isn't opt-in — it's the baseline.

02 Declare, don't configure

Say what you want, not how to render it. A NavigationDrawer takes data and pages — sections, dividers, transitions, breakpoints, and selection are handled internally. The library does the work.

03 Motion is meaning

Animation isn't decoration — it's communication. Every transition tells the user where they are and where they came from. Shared-axis page transitions, state layers, interaction feedback — all built in, all purposeful.

04 One source of truth

Colours, typography, motion, density — all from one theme singleton. Change a value once, every component updates. Globally. Instantly. No component defines its own sizes.

05 Performance is invisible

Transform-only animations. Gated GPU layers. No unnecessary redraws. The best optimisation is the one the user never notices because nothing ever stuttered.

The foundation.

Chi follows Dieter Rams' ten principles of good design — innovative, useful, aesthetic, understandable, unobtrusive, honest, long-lasting, thorough, environmentally friendly, and as little design as possible.

On top of that foundation, Chi implements Material Design 3 tokens — colour roles, typography scales, motion curves, and density tiers — adapted for the desktop-first context of Unix systems.

The result: components that feel both principled and alive.

Declare, Don't Configure

You write data. Chi builds the experience.

A navigation sidebar with sections, animated page transitions, and responsive breakpoints — in 15 lines.

main.qml 
import Chi

NavigationDrawer {
    items: [
        { section: "Mail", items: [
            { text: "Inbox",  icon: "inbox" },
            { text: "Sent",   icon: "send" },
            { text: "Drafts", icon: "edit" },
        ]},
    ]

    pages: [
        InboxPage  {},
        SentPage   {},
        DraftsPage {},
    ]
}
